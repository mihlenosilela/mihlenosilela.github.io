# Reflection of Coding My CV Using Markdown

## Situation

As part of Project Presentation 3, I was required to convert my traditional Word-based CV into a digital CV coded entirely using GitHub Flavoured Markdown. This was my first time working in a developer-focused environment using GitHub, so the experience felt both challenging and exciting.

## Task

My task was to learn Markdown syntax and apply it effectively to create a clean, readable, and professional CV suitable for viewing on GitHub Pages. The CV needed to clearly present my skills, education, and experience in a structured way.

## Action

I started by learning the basics of Markdown, including headings, lists, links, and spacing. I did my own research and watched YouTube tutorials to better understand how the language works.

I then restructured my CV into clear sections and removed unnecessary formatting from my original Word document. I focused on keeping the layout simple and easy to read. To ensure everything worked correctly, I regularly previewed my work both locally and on GitHub.

## Result

The final CV is clean, well-structured, and easy to navigate. This project improved my technical documentation skills and gave me practical experience using GitHub as a professional tool.

It also helped me understand how developers communicate using simple and structured formats. I now feel more confident using Markdown for documentation and building professional portfolios.

## Challenges & Solutions

One of the main challenges I faced was adjusting from a Word document to Markdown. Unlike Word, Markdown does not support advanced formatting, so I had to rethink how to present my information in a simpler way. I solved this by focusing on clarity and structure instead of design, using headings and spacing effectively.

Another challenge was remembering the correct Markdown syntax, especially at the beginning. I often made small mistakes with formatting, but I improved by practicing regularly and referring to online guides until it became more familiar.

Overall, these challenges helped me become more patient and detail-oriented, and they improved my ability to adapt to new tools and technologies.
